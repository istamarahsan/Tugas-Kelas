Pada code before ini terdapat smell refused bequest, yaitu sebuah smell yang dimana suatu class child menolak warisan atau tidak menggunakan seluruh method dari class parent. Terkadang bisa dikatakan juga bahwa kedua class tersebut tidak begitu memiliki relasi satu dengan yang lain sehingga method2 dari class parent tidak digunakan oleh class childnya.

Dalam kasus ini, pada code before dapat dilihat bahwa class company merupakan child dari class personalTaxCalculator, jadi class company mewarisi method2 dari class  personalTaxCalculator. Secara logika class company tidak ada hubungannya sama sekali dengan class personalTaxCalculator, satu2nya tujuan dari class company menginherit class personalTaxCalculator adalah untuk menggunakan 1 method saja yaitu method calculateTax. Hal ini dapat digolongkan sebagai smell refused bequest, karena pertama class company tidak ada hubungannya dengan class personalTaxCalculator sehingga method2 dari class parent tidak digunakan dengan baik banyak yang tidak terpakai karena tidak berhubungan.

Solusi yang kami berikan di code after adalah dari pada class company menginherit class  personalTaxCalculator, lebih baik dideclare saja class personalTaxCalculator di dalam class company (sebagai suatu object). Dengan begini maka code menjadi lebih masuk akal, karena tujuannya adalah hanya menggunakan suatu method dari class personalTaxCalculator pada class company maka lebih baik mendeclare sebuah object personalTaxCalculator saja daripada mengextend class personalTaxCalculator ke class company.
Dengan begini class company tidak mewarisi apa2 dari class apapun dan tentu saja tidak menolak method apapun dari class parentnya, dengan begitu smell refused bequest sudah hilang dari code ini.
