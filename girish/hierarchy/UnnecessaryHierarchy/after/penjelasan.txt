Unnecessary Hierarchy, pembagian class seharusnya dikaitkan dengan perbedaan behaviour bukan perbedaan variasi data yang digunakan. Pada kasus code before, dapat dilihat class shape di extends ke TwoDimensionalShape dan ThreeDimensionalShape baru 2 class tersebut diextends lagi ke class circle, cube dll. sebenarnya penggunaan hirarki ini tidak benar karena class dibagi berdasarkan data yang digunakan, kalau secara logika seharusnya ThreeDimensionalShape juga membutuhkan method menghitung perimeter, namun dikasus ini tidak digunakan makanya tidak di includekan methodnya pada ThreeDimensionalShape. 

Untuk fix smell pada kasus ini kami menghilangkan class TwoDimensionalShape dan ThreeDimensionalShape lalu kami memasukan method perimeter dan volume ke class shape, dengan begini kita telah mempersimple hirarki yang ada. Cara lain adalah kita memasukan method perimeter ke class shape namun method volume dideclare langsung di masing2 class yang menggunakannya seperti cube, dan Sphere.
