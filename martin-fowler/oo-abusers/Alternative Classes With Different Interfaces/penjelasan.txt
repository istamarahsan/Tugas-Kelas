Dalam kasus ini terdapat smell yaitu Alternative Classes With Different Interfaces.

Pada code before dapat dilihat terdapat 2 method pada 2 class yang berbeda yang memiliki fungsi yang mirip atau bahkan hampir sama yaitu method convertToCurrency pada class CheckoutHandler dan method convert pada class CurrencyConverter. Disini kami tidak bilang 2 method itu sama karena dari code before bawaan entah kami tidak tau itu kesalahan nulis atau tidak, namun ada sedikit perbedaan perkalian pada konveri ke IDR, kalau kami dapat berasumsi, kami rasa itu seharusnya sama nilainya yaitu antara 1 atau 15000  dikeduanya. Namun karena kami tidak ingin merubah behaviour code bawaan maka kami beranggapan saja bahwa 2 method tersebut mirip namun tidak sama. 

Jadi untuk menyelesaikan smell ini kami menghapus kedua method pada kedua class, lalu kami membuat sebuah map di main class. jadi kami memanfaatkan key value saja untuk logika konversi. karena pada dasarnya cara kerja konversi pada code before adalah bergantung dari input yang diberikan, input berupa mata uang. Mata uang tersebut kami buat enum dan kami jadikan key pada map yang kami buat, dan perkalian konversinya kami jadikan value. Namun disini kami bikin 2 map karena kami ingin tetap mempertahankan behaviour code before yang dimana kami tidak tau itu kesalahan nulis atau memang disengaja (perbedaan nilai konversi untuk mata uang IDR).

Dengan menggunakan map ini kami sudah berhasil menghilangkan smeel Alternative Classes With Different Interfaces, karena tidak ada lagi 2 method yang mirip di 2 class yang berbeda.
